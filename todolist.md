# ModelDownloader — 待办清单

## 一、需修复/完善的问题

### UI/UX

- [x] **深色主题适配不完整** — `ProgressDelegate` 中进度条背景/文字颜色改为使用调色板（`option.palette`），自动适配深浅色
- [x] **QProgressBar 样式硬编码** — 移除硬编码样式，使用系统默认主题
- [ ] **下载状态文字硬编码** — `STATUS_COLORS` 和 `ACTIVE_STATUS` 中的中文状态字典，语言切换时可能不匹配
- [x] **文件树对话框深色不适配** — 移除 `QTreeWidget` 硬编码颜色样式，`_guess_file_type` 改为 `tr()` 翻译
- [ ] **设置页无分组的 Card** — 外观/认证/关于卡片的 `QFrame` 未使用 `SettingCardGroup`，主题引擎可能不自动应用样式
- [ ] **镜像开关缺少独立卡片样式** — 与 `OptionsSettingCard` 样式不一致

### 功能缺陷

- [ ] **断点续传不完整** — `_wait_for_resume` 用忙等（`while _paused: time.sleep(0.1)`）；暂停后恢复是重头下载而非续传
- [x] **取消下载后文件未清理** — `DownloadTask._cleanup_file()` 在取消时删除临时文件
- [ ] **限速/队列并发控制简单** — `_max_concurrent = 3` 写死，不支持用户配置
- [ ] **无全局错误处理** — 网络错误、API 超时等未统一捕获弹窗提示
- [ ] **无下载完成通知** — 下载完成后没有系统托盘通知

### 代码质量

- [x] **`__init__.py` 为空** — 导出了 `cfg`, `tr`, `FileInfo`, `MainWindow` 等主要类
- [x] **`i18n.py` 字符串表维护麻烦** — 补充了文件类型、下载状态等 20+ 缺失翻译条目
- [x] **部分信号连接不严谨** — `cfg.language.valueChanged` 先用 `disconnect` 断开旧连接再绑定
- [ ] **硬编码字符串散落** — `"正在下载..."`、`"等待下载"` 等状态字符串在多个地方硬编码
